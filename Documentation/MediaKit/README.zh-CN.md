## 推荐发送顺序

1. 首次联系只发简短私信、GitHub 链接和 `social-card-zh-CN.png`。
2. 对方表示有兴趣后，再发两张实机截图、安装图和项目事实表。
3. 主动说明 Apple silicon、macOS 26 功能边界及当前未公证状态。
4. 邀请对方按实际体验决定是否收录，不要求照稿发布。

## 对外表述边界

推荐使用：

- “识别和可选翻译在本机完成”；
- “音频和导入稿件不会由应用上传”；
- “支持 Apple silicon，App 最低 macOS 15.5”；
- “Apple SpeechAnalyzer 和悬浮实时字幕需要 macOS 26”；
- “源码使用 MIT License；第三方组件和模型保留原许可”。

避免使用：

- “完全不联网”——模型下载、更新检查和外部文档需要联网；
- “所有 Mac 都支持”——暂不支持 Intel Mac；
- “完全 MIT”——可选 NLLB 模型为 CC-BY-NC-4.0；
- “准确率最高”——项目暂未发布统一基准测试；
- “macOS 15.5 支持全部功能”——Apple 实时字幕需要 macOS 26。
